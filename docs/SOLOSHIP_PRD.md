# Soloship 产品需求文档

> **定位**：让每个人都能拥有自己的 AI 员工团队，一个人也能运营一家公司。

---

## 一、产品愿景

Soloship 是一个 **2C 移动应用**。用户登录后，就是自己 AI 公司的 CEO——可以招募、配置和管理 AI 数字员工（Agent），让 Agent 7×24 小时替你工作，处理研究、写作、分析、任务执行等各种事务。

类比：就像你雇了一批专业助手，他们永远在线，有自己的性格、记忆和工作风格，不需要工资。

---

## 二、用户角色

| 角色 | 描述 |
|------|------|
| **用户（每个登录者）** | 默认拥有最高权限（platform_admin），是自己公司的老板 |
| **平台运营** | 通过后台直接操作数据库，不对外暴露 |

> **注意**：这是 2C 产品，没有多级企业管理员的概念。每个用户独立管理自己的空间。

## 四、技术架构

### 前端（Flutter App）

```
Flutter 3.x
├── 状态管理：Riverpod 2
├── 路由：GoRouter（含 /splash 启动页）
├── HTTP：Dio（Bearer Token 认证）
├── WebSocket：web_socket_channel
├── 认证：Firebase Auth（Google + Apple）
└── 本地存储：SharedPreferences（存 JWT token）
```

### 后端（Flutter 专用）

```
FastAPI + Python 3.12
├── 认证：Firebase ID Token 验证（RS256，无 firebase-admin SDK）
├── 数据库：PostgreSQL 15（端口 5434）
├── 缓存：Redis 7（端口 6380）
├── API 端口：8001
└── JWT 有效期：1 年
```

## 五、后续规划

### 第二阶段：虚拟办公室
- 像素 RPG 风格的虚拟办公室场景（Flame 引擎 + Tiled 地图）
- 每个 Agent 是一个可以行走的 NPC
- 用户角色可以在办公室里走动、和 Agent 对话
- Agent 之间可以开会、互动

### 第三阶段：移动端深度优化
- 手势操作（滑动、长按）
- 摇杆控制角色移动
- 推送通知（Agent 完成任务时提醒）
- 离线缓存

### 长期规划
- 多语言支持（英文、日文等）
- Agent 商店（下载别人分享的 Agent 模板）
- 团队协作模式（多个真实用户共享一个公司空间）
