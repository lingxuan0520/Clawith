## 六、成本优化建议

### 6.1 LLM 成本控制

| 策略 | 节省幅度 |
|------|---------|
| 日常对话用 Haiku/Sonnet，仅复杂任务用 Opus | 节省 80%+ |
| 设置每个 Agent 的每日 Token 上限 | 防止失控 |
| Heartbeat 统一使用 Haiku | Haiku 比 Opus 便宜 60x |
| 压缩 system prompt（精简 soul.md） | 降低每次调用成本 |
| 上下文窗口控制（`context_window_size`） | 历史消息适当截断 |

### 6.2 服务器成本控制

| 策略 | 节省幅度 |
|------|---------|
| Agent 容器按需启停（无活跃对话时停止） | 节省 60% 计算资源 |
| 对象存储替代本地文件系统（后期） | 按实际用量付费 |
| 使用竞价实例（抢占式）跑 Agent 容器 | 节省 50%~70% |
