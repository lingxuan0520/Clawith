# Soloship 产品需求文档（PRD）

> **版本**: v1.1
> **更新日期**: 2026-03-12
> **产品定位**: 2C 移动应用 — 让每个人拥有自己的 AI 员工团队

---

## 一、产品愿景

Soloship 是一个 **2C 移动应用**。用户登录后，就是自己 AI 公司的 CEO——可以招募、配置和管理 AI 数字员工（Agent），让 Agent 7×24 小时替你工作，处理研究、写作、分析、任务执行等各种事务。

**核心理念**：一个人也能运营一家公司。

类比：就像你雇了一批专业助手，他们永远在线，有自己的性格、记忆和工作风格，不需要工资。

---

## 二、用户角色

| 角色 | 描述 |
|------|------|
| **用户（每个登录者）** | 默认拥有最高权限（`platform_admin`），是自己公司的老板 |
| **平台运营** | 通过后台直接操作数据库，不对外暴露 |

> 这是 2C 产品，没有多级企业管理员的概念。每个用户独立管理自己的空间。所有 API 端点无角色校验。

## 三、功能需求

### 3.1 认证与账号

| 编号 | 功能 | 描述 | 优先级 | 状态 |
|------|------|------|--------|------|
| AUTH-01 | Google 登录 | Firebase Google Sign-In | P0 | ✅ 已完成 |
| AUTH-02 | Apple 登录 | Firebase Apple Sign-In（仅 iOS/macOS） | P0 | ✅ 已完成 |
| AUTH-03 | 自动注册 | 首次登录自动创建账号，默认 `platform_admin` 权限 | P0 | ✅ 已完成 |
| AUTH-04 | 保持登录 | JWT 有效期 1 年，启动时不闪登录页 | P0 | ✅ 已完成 |
| AUTH-05 | 头像同步 | 从 Firebase 同步头像（Google 有头像，Apple 无） | P1 | ✅ 已完成 |
| AUTH-06 | 首字母头像 | Apple 用户无头像时显示彩色首字母 | P2 | ⬜ 待做 |

**认证流程**：
1. App 启动 → `/splash` 加载页
2. 读取 SharedPreferences 中的 JWT → 调用 `GET /api/auth/me` 验证
3. 验证成功 → 跳转 `/plaza`；失败或无 token → 跳转 `/login`
4. 登录：社交登录 → Firebase → `POST /api/auth/firebase` → 返回平台 JWT

## 四、非功能需求

| 编号 | 类别 | 要求 |
|------|------|------|
| NFR-01 | 性能 | 页面加载 < 2 秒，聊天首字响应 < 1 秒 |
| NFR-02 | 可用性 | 支持 iOS 和 macOS（Flutter），后续支持 Android |
| NFR-03 | 安全性 | Firebase Auth + JWT 双重认证；API Key 加密存储 |
| NFR-04 | 国际化 | 当前阶段中文优先，预留 i18n 架构 |
| NFR-05 | 数据隔离 | 每个用户（Tenant）数据完全隔离 |
| NFR-06 | Agent 隔离 | 文件目录隔离 + 数据库 Tenant ID 过滤，工具调用链共享 |
